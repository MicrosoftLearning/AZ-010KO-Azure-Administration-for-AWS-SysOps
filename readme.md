## GitHub 소개

- Microsoft는 이 과정을 지원하기 위해 과정 콘텐츠를 수시로 업데이트하여 과정 내에서 사용되는 Azure 서비스 관련 정보가 적용된 최신 상태로 유지해야 합니다.  따라서 Azure 플랫폼의 변경 내용을 적용하여 콘텐츠를 최신 상태로 유지하기 위해 과정 작성자와 MCT가 자유롭게 교환할 수 있도록 GitHub에 랩 지침과 랩 파일을 게시하고 있습니다.

- GitHub를 활용하면 새로운 공동 작업 방식으로 랩을 진행할 수 있습니다. 즉, Azure를 실제로 사용하는 중에 변경된 부분을 먼저 확인하는 작성자/MCT가 랩 원본을 바로 개선하여  동료 MCT에게 도움을 줄 수 있습니다.

## 릴리스된 MOC 파일과 함께 이러한 파일을 사용할 수 있는 방식

- 기존 방식대로 강사용 핸드북과 PowerPoint를 과정 콘텐츠 교육용 기본 원본으로 계속 활용할 수 있습니다.

- GitHub에서 제공되는 이러한 파일은 학생용 핸드북과 함께 사용할 수 있으며, GitHub의 중앙 리포지토리에 포함되어 있으므로 MCT와 과정 작성자에게 최신 랩 파일의 공유 원본이 제공됩니다.

- 교육 담당자는 교육을 진행할 때마다 GitHub에서 최신 Azure 서비스를 지원하기 위해 적용되었을 수 있는 변경 내용을 확인해야 하며, 교육 시 활용할 최신 파일을 다운로드해야 합니다.

## 학생용 핸드북의 변경 방식

- Microsoft에서 분기별로 학생용 핸드북을 검토하고 필요에 따라 정규 MOC 릴리스 채널을 통해 핸드북을 업데이트할 예정입니다.

## 콘텐츠 제공 방법

- 모든 MCT는 GitHub 리포지토리의 코드나 콘텐츠에 대해 끌어오기 요청을 제출할 수 있습니다. 그러면 Microsoft와 과정 작성자가 콘텐츠를 심사한 다음 필요에 따라 콘텐츠/랩 코드 변경 내용을 포함합니다.

- 버그, 변경 내용, 개선 사항 및 아이디어를 제출할 수 있습니다.  새로운 Azure 기능을 찾으셨다면  새로운 데모를 제출해 보세요!

## 참고 사항

### 강의용 자료

MCT와 파트너는 이러한 자료를 확인한 다음 학생들에게 별도로 제공하는 것이 좋습니다.  학생들이 진행 중인 강의의 일환으로 GitHub에서 직접 랩 단계에 액세스하도록 안내하는 경우, 과정의 일부분으로 또 다른 UI(GitHub의 UI)에 액세스해야 하므로 혼선이 발생할 수도 있습니다. 따라서 학생들에게 별도의 랩 지침을 제공하는 이유를 설명할 때는 클라우드 기반 인터페이스와 플랫폼이 지속적으로 변화하기 때문임을 강조할 수 있습니다. Microsoft Learning에서는 이 과정의 교육을 진행하는 MTC만 GitHub의 파일에 액세스하고 GitHub 사이트를 탐색할 수 있도록 지원합니다.
